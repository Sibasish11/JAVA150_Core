# Write a program to simulate a race condition where multiple threads try to increment a shared counter without synchronization. Observe the incorrect results. [Medium]

```java

- Race conditions occur when multiple threads access and modify shared data
  concurrently without proper synchronization.
- In this example, multiple threads increment a shared counter, but due to
  the lack of synchronization, the final result may be incorrect.

Expected result:
- If 1000 increments are performed, the final counter should be 1000.
- Without synchronization, you’ll often see a lower value (because threads
  overwrite each other’s updates).


class SharedCounter {
    int count = 0; // shared variable

    public void increment() {
        count++;
    }
}

class CounterThread extends Thread {
    SharedCounter counter;

    CounterThread(SharedCounter counter) {
        this.counter = counter;
    }

    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }
}

public class RaceConditionDemo {
    public static void main(String[] args) {
        SharedCounter counter = new SharedCounter();

        // Create multiple threads
        CounterThread t1 = new CounterThread(counter);
        CounterThread t2 = new CounterThread(counter);
        CounterThread t3 = new CounterThread(counter);

        t1.start();
        t2.start();
        t3.start();

        try {
         
            t1.join();
            t2.join();
            t3.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        // Display final counter value
        System.out.println("Final counter value (expected 3000): " + counter.count);
    }
}

```
