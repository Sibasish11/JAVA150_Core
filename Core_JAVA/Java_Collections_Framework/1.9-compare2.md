# Compare and contrast HashMap, LinkedHashMap, and TreeMap. Demonstrate their behavior with a few entries. [High]

```java

import java.util.HashMap;
import java.util.LinkedHashMap;
import java.util.TreeMap;

public class MapComparison {
    public static void main(String[] args) {
        // HashMap - No order guaranteed
        HashMap<String, Integer> hashMap = new HashMap<>();
        hashMap.put("Banana", 2);
        hashMap.put("Apple", 5);
        hashMap.put("Mango", 3);
        hashMap.put("Orange", 4);

        // LinkedHashMap - Maintains insertion order
        LinkedHashMap<String, Integer> linkedHashMap = new LinkedHashMap<>();
        linkedHashMap.put("Banana", 2);
        linkedHashMap.put("Apple", 5);
        linkedHashMap.put("Mango", 3);
        linkedHashMap.put("Orange", 4);

        // TreeMap - Maintains keys in sorted order
        TreeMap<String, Integer> treeMap = new TreeMap<>();
        treeMap.put("Banana", 2);
        treeMap.put("Apple", 5);
        treeMap.put("Mango", 3);
        treeMap.put("Orange", 4);

        // Printing all
        System.out.println("HashMap:       " + hashMap);
        System.out.println("LinkedHashMap: " + linkedHashMap);
        System.out.println("TreeMap:       " + treeMap);
    }
}

/*
==================== CONCEPTUAL COMPARISON ====================

1. Internal Implementation:
   - HashMap: Uses a hash table (array + linked list/tree for collisions).
   - LinkedHashMap: Extends HashMap with a doubly-linked list to maintain order.
   - TreeMap: Implements NavigableMap, uses Red-Black Tree for sorted ordering.

2. Ordering:
   - HashMap: No guarantee of order.
   - LinkedHashMap: Maintains insertion order.
   - TreeMap: Maintains keys in natural ascending order (or custom Comparator).

3. Performance:
   - HashMap: O(1) average for put/get.
   - LinkedHashMap: O(1) average for put/get, with slight overhead for order tracking.
   - TreeMap: O(log n) for put/get due to tree structure.

4. Null Keys/Values:
   - HashMap: One null key allowed, multiple null values allowed.
   - LinkedHashMap: Same as HashMap.
   - TreeMap: Null keys not allowed (NullPointerException if used), null values allowed.

5. Use Cases:
   - HashMap: Best for fast lookups with no order requirement.
   - LinkedHashMap: When you need predictable iteration order.
   - TreeMap: When you need sorted keys or range queries.
*/


```
